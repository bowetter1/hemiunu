# Spec Template

> Kopiera och fyll i för varje uppgift.

---

## UPPGIFT: [Kort titel]

### Roll
[Vilken roll ska utföra detta?]

### Kontext
[1-2 meningar: Vad? Varför? Var i projektet?]

---

### Input

**Filer att läsa:**
- `path/to/file`

**Beroenden:**
- [Andra specs/filer som måste vara klara först]

---

### Implementation

1. [Steg 1]
2. [Steg 2]
3. [Steg 3]

---

### Output

**Förväntat resultat:**
- [ ] [Fil/artefakt som ska skapas]

**Gör INTE:**
- [Saker utanför scope]

---

### Acceptanskriterier

- [ ] [Kriterium 1]
- [ ] [Kriterium 2]

---

## Exempel

```markdown
## UPPGIFT: Designa färgpalett

### Roll
AD

### Kontext
Vi bygger en pyramid-app. Behöver mörkt tema med accent-färger.

### Input
- Kolla competitors: [länk], [länk]
- Läs wireframes.md för kontext

### Implementation
1. Surfa inspiration
2. Välj primär, sekundär, accent
3. Testa kontrast
4. Dokumentera i design_system.md

### Output
- [ ] design_system.md med färgsektion

### Acceptanskriterier
- [ ] Minst 4 färger definierade
- [ ] Kontrasten fungerar för läsbarhet
```
